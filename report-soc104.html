SOC104 — Malware Detected:
WinRAR Installer

LetsDefend Medium Severity False Positive Threat Intel Hash Analysis

Summary

An alert flagged winrar600.exe on host SusieHost (172.16.17.5) as potential malware. Thorough investigation across VirusTotal, Hybrid Analysis, and LetsDefend TI returned clean results across the board. Log analysis showed no suspicious behavior. A notable finding was the offline EDR agent — flagged as a separate visibility gap requiring remediation. The alert was correctly closed as a False Positive.

Investigation Walkthrough

1. Initial Alert Review

Documented hostname (SusieHost), source IP (172.16.17.5), filename (winrar600.exe), file hash, and timestamp. The filename appeared to reference a legitimate WinRAR installer, but full validation was required before drawing any conclusions.

2. File Hash Analysis — VirusTotal

Submitted the file hash to VirusTotal. Result: 0 detections across all security vendors. No malicious signatures, behaviors, or community flags associated with the hash.

3. Behavioral Analysis — Hybrid Analysis

Submitted to Hybrid Analysis for dynamic behavioral analysis. File was returned as whitelisted — matched a known safe application profile. No suspicious process, network, or file system activity flagged.

4. Log Analysis

Reviewed system logs and identified a single download event for winrar600.exe. The download URL was checked on VirusTotal — returned clean. No suspicious log entries, unusual process executions, or abnormal network connections observed.

5. EDR Analysis — Visibility Gap Found

Attempted endpoint analysis via EDR but found the EDR agent on SusieHost was offline. This was flagged as a separate critical finding — a visibility gap that must be remediated regardless of the alert outcome.

⚠ Even though this alert is a false positive, the offline EDR agent is a real security risk that was flagged and escalated independently.

6. Why This Alert Fired

The alert was likely triggered by a heuristic or behavioral rule reacting to an external .exe download — expected behavior for security tools that err on the side of caution. The SOC analyst's role is to validate these alerts through evidence, not assumptions.

MITRE ATT&CK Mapping

No MITRE ATT&CK techniques were identified. There is no evidence of malicious activity associated with this alert.

Verdict
Final Classification
False Positive — Legitimate Software
Recommendations
  • Investigate and restore the offline EDR agent on SusieHost — endpoint visibility is critical
  • Whitelist the confirmed file hash to reduce future false positive alerts
  • Ensure all endpoints have active, healthy EDR agents — run a systematic check
  • Review detection rules that triggered this alert to reduce noise for known-safe installers
  • Implement an approved software list to distinguish sanctioned downloads from unauthorized ones
Key Learnings
  • Not every "Malware Detected" alert is real. Correctly identifying false positives is just as important as catching threats — it keeps the team focused and reduces alert fatigue.
  • Multi-source TI builds confidence in both directions. When VirusTotal, Hybrid Analysis, and LetsDefend TI all agree a file is clean, that convergence is meaningful evidence.
  • Visibility gaps must be flagged. The offline EDR agent was a separate significant finding — even if the current alert is benign, the lack of endpoint visibility is a real risk.
  • Evidence-based decisions over assumptions. The job of a SOC analyst is to follow the evidence — in this case, it clearly pointed to a legitimate file.
  • False positives have real cost. Misclassifying a false positive wastes response resources and erodes team trust in the alerting system. Accurate classification is a core SOC skill.
