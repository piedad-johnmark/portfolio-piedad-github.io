Summary

A complete ransomware attack chain was reconstructed using PCAP network traffic and Windows EVTX event logs. The attacker began with active port scanning, identified an exposed MSSQL server, authenticated with valid credentials, enabled xp_cmdshell for RCE, escalated privileges via process injection into winlogon.exe, disabled Windows Defender, established persistence via scheduled task, dumped credentials, mapped the network, and finally deployed BlueSky ransomware (javaw.exe) — resulting in file encryption and ransom note creation.

Attack Chain Reconstruction
1
RECONNAISSANCE
Port Scanning
Wireshark filter tcp.flags.syn == 1 revealed TCP SYN requests across multiple ports. Protocol Hierarchy showed TDS traffic — MSSQL server identified as target.
2
INITIAL ACCESS
MSSQL Credential Abuse
TDS filter located a TDS7 login packet with valid credentials used to authenticate to the MSSQL server. Confirmed using valid account abuse as initial access vector.
3
EXECUTION
xp_cmdshell Enabled
Attacker executed SQL commands to enable xp_cmdshell — turning the database into a remote code execution engine. Confirmed in both PCAP and EVTX logs (dual-source validation).

EXEC sp_configure 'xp_cmdshell', 1; RECONFIGURE;
4
PRIVILEGE ESCALATION
Process Injection → winlogon.exe
PowerShell EVTX logs confirmed process injection into winlogon.exe — a SYSTEM-level process that's always running, giving the attacker SYSTEM privileges and a persistent C2 channel.
5
DEFENSE EVASION
Windows Defender Disabled
HTTP filter GET revealed checking.ps1 download. Stream analysis showed it checked Group SID and modified registry keys to disable Windows Defender. del.ps1 provided additional evasion.
6
PERSISTENCE
Scheduled Task Created
Filter http contains "schtasks" + TCP stream follow revealed full task name: \Microsoft\Windows\MUI\LPUpdate — disguised as a legitimate Windows path.
7
CREDENTIAL ACCESS
Invoke-PowerDump.ps1
Stream analysis revealed Invoke-PowerDump.ps1 usage. An EncodedCommand was decoded via CyberChef (Base64) — output file identified as hashes.txt containing dumped credential hashes.
8
DISCOVERY
Network Host Enumeration
Stream analysis revealed extracted_hosts.txt — a list of internal systems discovered, indicating preparation for lateral movement.
9
IMPACT
BlueSky Ransomware Deployed
Filter http contains ".exe" identified javaw.exe download from 87.96.21.84. Initially clean on VirusTotal/Talos — only ANY.RUN sandbox confirmed it as BlueSky ransomware, resulting in file encryption and ransom note creation.

Key Learnings
  • PCAP + EVTX correlation is powerful. The same SQL commands appeared in both network traffic and event logs — dual-source confirmation removes all doubt.
  • xp_cmdshell is a critical MSSQL hardening target. When enabled, it turns a database into a full RCE platform. It should always be disabled in production environments.
  • TI can miss new ransomware samples. javaw.exe was clean on VirusTotal and Cisco Talos initially — only ANY.RUN sandbox confirmed it as BlueSky. Never rely on a single clean verdict for executables.
  • Wireshark filters are essential for efficiency. tcp.flags.syn==1, tds, http.request.method=="GET", and http contains "schtasks" dramatically narrowed relevant traffic in a large PCAP.
  • Base64 encoding is a common obfuscation technique. Decoding attacker commands with CyberChef is a fundamental analyst skill for uncovering what malicious scripts are actually doing.
  • Reconstructing the full attack chain matters. Understanding how each stage connects — from port scan to ransomware — separates reactive alert handling from proactive incident response.
